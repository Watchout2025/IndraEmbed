Capabilities

Everything the player already does, before you write a line of code.

Each card below is a feature already shipping in Vidout's core build — not a roadmap item.

Adaptive HLS Engine

Built on HLS.js with custom error-recovery logic, so a stalled manifest or dropped segment doesn't end the session.

Frame-Accurate Subtitles

Rendering mirrors the DOM and inline-style structure used by top multi-platform streaming services — positioning, line breaks, and timing included.

Episode Intelligence

Season tabs, a scrollable episode rail, and a NOW badge that tracks exactly where the viewer left off.

Intro & Recap Skip

IntroDB-powered skip segments let viewers jump straight past the part of the episode they've already seen.

Drag-and-Drop Local Playback

Drop a local file onto the player and it routes straight to blob URL or HLS playback — no upload step.

One File, Any Host

The entire player lives in a single HTML build. Deploy it to Cloudflare Pages, Netlify, or just a folder.

Continue Watching

Playback position saves automatically as it plays, so reopening a title resumes from that exact timestamp — no manual seeking.

Quick Start

Deploy your own copy, then embed it anywhere.

Vidout isn't a shared public service — it's a single HTML file you host yourself. Once it's live on your own domain, embedding it is the easy part.

1

Deploy your own copy

Push the single HTML build to Cloudflare Pages, Netlify, or any static host you control.

2

Find the TMDB ID

Search any title on TMDB and grab the numeric ID from its URL.

3

Point the iframe at your deployment

Swap in your own domain below. Movie and TV paths, plus ?url= for a direct source, all resolve automatically.

embed.html 
<!-- Replace with the domain where you've deployed your own copy -->
<iframe
  src="https://your-deployment.pages.dev/embed/movie/299534"
  width="100%"
  height="100%"
  allow="autoplay; fullscreen"
  frameborder="0">
</iframe>
TV paths accept either format: /tv/{id}/S1/E2 or /tv/{id}/1/2 — both resolve to the same episode.
Skip the TMDB lookup entirely with ?url= — point it at any HLS source directly: /embed?url=https://example.com/stream.m3u8
